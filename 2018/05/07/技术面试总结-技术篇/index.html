技术面试总结-技术篇

编程思想

面向过程：典型的是C/C++的结构体，结构体里只有变量，没有处理变量的方法，需要专门编写处理变量的方法。
面向对象：ArrayList list=new ArrayList();坏处是如果改为LinkedList，所有代码可能需要重写，同时扩展一个新的List的话，需要重新调用。
面向接口：List=new Arraylist();是面向对象的一种形式，广义上讲，抽象类也是接口的一种形式，使用该类的某种属性（接口）来表示，降低耦合性，增加代码复用性。
面向切面：这种在运行时，动态地将代码切入到类的指定方法、指定位置上的编程思想就是面向切面的编程。

AspectJ的使用及原理

1、需要登录后的操作
2、防止快速点击操作
3、异常捕获及处理
Spring实现的AOP是代理模式,给调用者使用的实际是已经过加工的对象,你编程时方法体里只写了A,但调用者拿到的对象的方法体却是xAy。
备注：AspectJ的AOP是编译时织入class文件，打包时已经修改了源代码。

性能优化

布局优化

1、减少界面层级(比如使用：Hierarchy Viewer、ViewServer工具)
2、减少过度渲染（多是界面重叠的情况，list使用type、多activity）
3、使用include、viewstub、merge减少层级

内存优化

1、图片加载限制大小
2、leakcanary检查内存泄露

网络优化

0、网络分析工具，Android Studio Network Monitor;抓包工具Charles==
1、网络缓存（省流量）
2、加载周期监听，及时响应（提升用户体验）
3、网络图片质量、尺寸定制请求（省流量）
4、弱网情况（模拟器、代理模拟），使用离线缓存

稳定性

1、全局异常捕获（catch、loop），传送到后台分析
2、多进程，保证主进程不阻塞

内存泄露的原因

1、全局静态变量直接/间接（view）持有上下文对象
2、activity进程外的线程持有activity强引用
3、内部类、匿名内部类持有外部类引用
4、定时器

如何做好可扩展性，重构和设计好的接口？
1、好的设计不是设计出来的，是改出来的
一开始所谓的高扩展性，会逐渐因为需求的不对称而放弃。随着业务需求的不断修改，以及扩展需求的改进，做好长期进化设计的准备。
举例：adapter进化、列表加载更多自动化实现
2、面向接口编程（里氏替换原则）
举例：application抽象接口appDelegate
3、单一职责原则
举例：[点这里]（https://blog.csdn.net/yanbober/article/details/45312243）

M：Retrofit2（网络）+Ormlite（数据库）+Glide/Fresco（图片加载）
V：多Activity+多Fragment+EventBus+列表加载更多自动化
P：业务逻辑处理

Retrofit2:动态代理包装。我就是一个包含了rxjava和okhttp的壳而已
RxJava:一个带有目的（行为计划）的报刊（可订阅者）被订阅的故事
EventBus:订阅者模式，反射解析、线程切换执行。 处处埋伏，只为找到对的你

ReactNative之坑

1、组件在ios、android两个平台有一定的差异性，比如：DatePickerIos
解决方法：差异处理，或者使用三方库来处理
2、不能兼容web前端
解决方法：如果要在web端运行，需要重新写一套代码。
或者使用现有框架来进行转换，把ReactNative代码转换成ReactJs代码进行部署，比如：携程Moles，京东JDReact
3、不支持热更新
解决方法：使用三方库，比如微软codepush
4、性能瓶颈，尤其是交互非常复杂、页面非常复杂、需要频繁的更新、需要一些手势交互的场景

性能问题解决方案：
1、分片（自定义组件）加载，利用状态与组件的绑定关系
2、shouldComponentUpdate来判定组件是否要加载
3、关闭测试的log

handler机制

2、handler向looper中消息列队添加消息
3、looper中消息队列轮询处理消息

内存泄露检查工具leakcanary原理

1、通过ActivityLifecycleCallbacks监听activity弱引用实例，还可以监听Fragment
2、当页面关闭时，调用一次GC，RefWatcher检查实例是否被回收
3、通过haha库分析内存泄露调用链，并显示通知

android IPC

IPC是 Inter-Proscess Communication的缩写，含义为进程间的通讯或者跨进程通讯
Bundle、文件共享、AIDL、Messenger、ContentProvider==

AIDL使用和原理：

1、跨进程通信，服务端/客户端
2、服务端/客户端创建aidl文件，实现stub抽象类，客户端绑定service，调用aidl接口
原理：代理模式
1、服务端创建一个Service用来监听客户端的连接，生成代理接口、实现代理类，通过bind service来传递代理对象
2、客户端实现service回调来拿到服务端的代理对象
3、客户端通过代理接口调用服务端的功能

NDK使用

1、c调用java，使用jni反射API调用java代码
2、java调用c，定义native函数调用c代码
3、NDK生成so文件并打包到apk运行

NDK 好处：

1、NDK 可以生成 .so 文件, 方便代码共享；
2、增加反编译的难度；
3、提高性能。

ali题

1、view的加载过程
2、Java内存模型是什么？
Java内存主要分三部分：一部分是线程共享的，包括 Java 堆和方法区，堆Heap、栈Stack、方法区。
Heap:大部分实例对象占用的空间
Stack:虚拟机Stack（方法帧栈）和本地虚拟机Stack（本地JNI帧栈）
方法区:类型数据、静态变量实例、常量
